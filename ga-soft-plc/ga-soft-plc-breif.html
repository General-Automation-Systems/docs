Общая информация

GA Soft PLC - это универсальное программное обеспечение, предназначенное для автоматизации технологических процессов, позволяющее осуществлять управление конвейерными, сортировочными, шатловыми системами и сопутствующим периферийным оборудованием. Ключевой особенностью GA Soft PLC в сравнении с классическими PLC является отсутствие привязки потребителя к вендору производителю аппаратной части PLC, т.е. GA Soft PLC имеет минимальные требования к аппаратной части и фактически может быть развернуто как на стационарном PC так и на промышленном компьютере общего назначения. GA Soft PLC является одним из компонентов GA TCS в которую также входят:

  • GA Roller Controller - контроллер приводных устройств конвейера
  • GA Signal Router - Сетевой "роутер" для объединения двух групп контроллеров приводных устройств

Архитектура GA Soft PLC

На верхнем уровне GA Soft PLC представляет из себя набор сервисов для интеграции в инфраструктуру склада некоторые из которых , а именно:

  • WMS (Warehouse Management System ) - API для интеграции с WMS. Возможны разные варианты интеграции: интеграция имеющегося API заказчиком, разработка API под требования заказчика.
  • HMI (Human-machine interface ) - В общем случае HMI представляет из себя визуализацию состояния автоматизированного тех. процесса/конвейера/сортировочной системы. В GA Soft PLC визуализация представляется Web приложением доступным из браузера с любого устройства. По желанию заказчика может быть перенесена на любую HMI панель.
  • HTTP сервер - сервер с открытым API для интеграции с другими сервисами и службами на стороне заказчика.

Ядро системы представляет из себя ОС реального времени с необходимым набором ПО для управления тех процессом посредством нижележащих уровней абстракции.

Преимущества GA Soft PLC

Описание преимуществ использования GA Soft PLC в сравнении с традиционными PLC:

  • Отсутствие вендор-локов (нет привязки разработанного ПО к конкретному вендору PLC, возможен перенос ПО на широкую номенклатуру разных производителей аппаратной части)
  • Сокращение простоев оборудования.
  • Гибкая архитектура ПО с возможностью интеграции и доработки под любые системы и сервисы заказчика.
  • Постоянная поддержка и удаленное обновление.
